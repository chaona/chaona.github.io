JavaScript诞生于1995年，是为了解决表单数据输入验证。

当时的网速很慢，频繁的表单验证与服务器交换数据只会加重用户的负担。

JavaScript 从一个简单的输入验证器发展成为一门强大的编程语言，完全出乎人们的意料。

JavaScript组成

  • 核心（ECMAScript），提供核心语言功能；
  • 文档对象模型（DOM），提供访问和操作网页内容的方法和接口；
  • 浏览器对象模型（BOM），提供与浏览器交互的方法和接口。

核心ECMAScript

ECMAScript是一种标准，是JavaScript实现的基础。

文档对象模型DOM

文档对象模型（DOM，Document Object Model）是针对 XML 但经过扩展用于 HTML 的应用程序编 程接口（API，Application Programming Interface）。

DOM 把整个页面映射为一个多层节点结构。HTML 或 XML 页面中的每个组成部分都是某种类型的节点，这些节点又包含着不同类型的数据。


<html>
    <head>
        <title>Sample Page</title>
    </head>
    <body>
        <p>Hello World!</p>
    </body>
</html>

通过 DOM 创建的这个表示文档的树形图，开发人员获得了控制页面内容和结构的主动权。借助 DOM 提供的 API，开发人员可以轻松自如地删除、添加、替换或修改任何节点。

为什么要使用DOM？开发人员首次无需重新加载网页，就可以修改其外观和内容了。

浏览器对象模型BOM

浏览器对象模型（BOM，Browser Object Model）。开发人员使用 BOM 可以控制浏览器显示的页面 以外的部分。

从根本上讲，BOM 只处理浏览器窗口和框架；但人们习惯上也把所有针对浏览器的 JavaScript 扩展 算作 BOM 的一部分。

使用JavaScript

在HTML中使用JavaScript的方法，就是使用 <script>元素：

  • 内嵌式
  • 外链式

<script>元素的位置一般放置</body>标签之前。

JavaScript文件命名规范:全部采用小写方式，以下划线分隔。

例：account_model.js

标识符

所谓标识符，就是指变量、函数、属性的名字，或者函数的参数。

ECMAScript中的一切（变量、函数名和操作符）都区分大小写。

变量命名规范

  • 标准变量采用小驼峰式命名：var bookName
  • 'ID'在变量名中全大写
  • 'Android'在变量名中大写第一个字母
  • 'iOS'在变量名中小写第一个，大写后两个字母
  • 常量全大写，用下划线连接:var MAX_COUNT = 100;
  • jQuery对象必须以'$'开头命名：var $body = $('body');

虽然没有谁强制要求必须采用这种格式，但为了与 ECMAScript 内置的函数和对象命名格式保持一 致，可以将其当作一种最佳实践。

注释

ECMAScript 使用 C 风格的注释，包括单行注释和块级注释。实际工作中还会用到：文档注释

单行注释

双斜线后，必须跟一个空格；

缩进与下一行代码保持一致；

可位于一个代码行的末尾，与代码间隔一个空格。

if(condition) {
    // 单行注释
    allowed();
}

var bookName = 'JavaScript高级编程'; // 单行注释

多行注释（块级注释）

最好三行，'*'后跟一个空格。

使用场景：

  • 难以理解的代码段
  • 可能存在错误的代码段
  • 浏览器特殊的HACK代码
  • 业务逻辑强相关的代码
/*
 * one space after '*'
 * 这是一个多行（块级）注释
 */
var x = 1;

文档注释

各类标签@param，@method等。

使用场景：

  • 所有常量
  • 所有函数
  • 所有类
/**
 * @func
 * @desc 一个带参数的函数
 * @param {string} a - 参数a
 * @param {number} b=1 - 参数b默认值为1
 * @param {string} [c] - 参数c是一个可选参数
 */
function foo(a, b, c) {
    // code here
    ...
}
